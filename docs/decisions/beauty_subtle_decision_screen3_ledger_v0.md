## Procedure Findings

- DRY RULE FIRST FIRING + BREACH: agent 1's moves 1-2 were both DRY (two
  consecutive) — the stop should have fired at move 2 and the walker continued
  to move 12. Execution breach, honestly recorded. The breach exposed a real
  design ambiguity the rule should name: ACCESS-DRY (the query/tool shape
  returned nothing — moves 1-2 were search-index failures, not walked venues)
  vs VENUE-DRY (a venue was walked and yielded nothing). Counting access
  failures toward the dry stop would have killed a walk that went on to
  produce 4 fragments and 2 detector observations. Proposed-not-applied:
  amend the dry rule by dated note to count only venue-dry moves, with
  access-dry recorded as an access-shape note. OWNER-GATED.
- Reddit-index access gap (structural): the origination layer's Reddit venues
  were unreachable via web-search indexing this run despite multiple query
  shapes. The functional detection layer that worked instead: audit blogs +
  Basenotes snippets. Any future subtle-class screen needs a Reddit read
  shape that does not depend on external search indexing (in-screen direct
  reads); routing that question to the capture-recon seam is a screening-read
  question, not capture.
- Silent-reformulation A-leg gap (structural): for the purest subtle class —
  the brand NEVER acknowledges (Xerjoff) — the assembly model's A-leg (brand
  action evidence) may be structurally unavailable. Proposed-not-applied
  refinement: allow the A-leg to be satisfied by MATERIAL CHANGE evidence
  (INCI diffs, batch/packaging-period correlation, before/after archived
  pages) when no brand statement exists. OWNER-GATED.
- Assembly model otherwise held: fragments-as-evidence kept walks alive and
  honest; completions were the cheapest moves and both resolved (compounding
  visible: screen 2's held-back fragments became screen 3's first yields).
- Caps held (12/12 and 12/12); candidate-stop never reached; agent 2's dry
  rule never fired two-consecutive.
