## 2. Second-Pass Rationale

The portfolio is selected to avoid validating only one type of judgment.

The set must test both:

- upside recognition: whether Orca can justify a reliable `Test`, bounded
  `Move`, partnership, acquisition, or build path before the later outcome is
  obvious;
- mistake avoidance: whether Orca can block, delay, narrow, or downgrade a move
  that later performs badly.

This matters commercially because a buyer will not pay only for caution. The
method must show that it can help teams act sooner when evidence convergence is
strong enough, and also help teams avoid expensive false moves when the public
evidence is polluted.

## 5. Required Locks Before Evidence Replay

Before any evidence collection or replay begins, each locked case still needs:

- exact decision question;
- decision owner or context;
- cutoff date or cutoff window;
- fair-cutoff rationale;
- first-order source-family boundary;
- second-order source-family boundary;
- post-window exclusion rule;
- expected later outcome comparison window;
- action verbs allowed for the case;
- costly behavior or reliable-bet standard for the domain;
- reverse-case or positive-action role, where applicable;
- pass, fail, blocked, inconclusive, useful, wrong, early, late,
  overconfident, and underconfident result semantics.

No case may be replayed until these locks are recorded.
