## 2. Discovery Boundary Actually Used

The pass asked only eligibility questions:

- does the candidate exist;
- did the pre-outcome pressure moment occur between 2022-01-01 and
  2026-05-21;
- is there a plausible fair cutoff before the later outcome was obvious;
- was there public pre-cutoff visibility;
- is a later public outcome already visible by 2026-05-21;
- can post-window material be excluded from the at-cutoff replay;
- is public inspectability plausible for a later leakage-controlled backtest.

The pass did not ask whether pre-cutoff signals were strong, whether buyers
changed behavior, what Core Spine would recommend, whether the backtest would
pass, or whether any outcome proves Core Spine v0 works.

## 3. Evaluation Criteria

An AI-era backtest candidate is stronger for the first proof run when it has:

- a clean AI-era pressure moment after 2022-01-01;
- a later outcome visible before 2026-05-21;
- official or otherwise inspectable public surfaces on both sides of the
  cutoff;
- a cutoff that can be frozen without flattering hindsight;
- enough decision ambiguity at cutoff to preserve wrong, early, late,
  underconfident, overconfident, blocked, and inconclusive outcomes;
- limited need for ordinary-person evidence, private data, analyst-only
  evidence, source maps, or runtime collection;
- a decision context close to product, positioning, pricing, growth,
  platform, partnership, acquisition, retreat, or buyer-facing AI workflow
  change.
